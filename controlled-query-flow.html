Controlled Query & Hierarchical Routing — 无人机网络身份体系示意图

0
身份准备（无人机端）
Identity Preparation
BLOCK A

网络身份 ID 创建

UOM 委派 RAA/HDA 地址段给云平台。无人机获得包含层级前缀的 DET（网络标识符）。

RAARoot Authority Agent
HDAHierarchical Domain Agent
Hash 指纹公钥哈希 (cSHAKE128)

▲ DET 结构：RAA 前缀 + HDA 前缀 + 公钥哈希指纹

BLOCK B

身份凭证生成

🔑
Ed25519 密钥对 — 本地生成
HHSI 声明 — 哈希和算法标识符
cSHAKE128 — 计算公钥哈希

无人机在本地生成 Ed25519 密钥对和 HHSI 声明，计算公钥的 cSHAKE128 哈希。该哈希指纹成为 DET 的组成部分，作为身份验证基石。

1
查询发起与接收（统一网关）
Query Initiation & Gateway
BLOCK C

查询方发送查询请求

查询方（公众或监管主体）发送包含特定无人机 DET 的查询请求，请求到达统一网关。

查询方类型：公众用户 (Public) ｜ 监管/应急主体 (Regulatory)
BLOCK D

统一网关（路由器/解析器）

  • 解析标识位段：从 DET 中提取 RAA 和 HDA 前缀
  • 执行路由匹配：基于 HDA 前缀（如 00B2）确定管理节点
  • 无需全表扫描：直接前缀定位，高效路由
RAA ✓提取
HDA ✓路由匹配键
Hash此阶段不使用

Hierarchical Routing

基于 HDA 前缀的直接路由

统一网关依据 HDA 前缀，将查询请求无需全表扫描，直接路由/重定向到对应的管理节点。

路由逻辑：Gateway 解析 DET → 提取 RAA & HDA 段 → 依据 HDA 前缀（如 00B2）重定向 → 目标管理节点（HDA2），全程无需全表扫描。
3
管理节点：数据提取与视图切片
View Slicing & Access Control
BLOCK E

HDA 管理服务器接收查询

HDA2 管理服务器接收查询，查找无人机的全量数据：

  • 精确位置信息
  • 运行状态数据
  • 所有人身份信息
  • 授权与证书信息
BLOCK F

差异化授权与视图切片

在数据返回前，HDA 执行视图切片：

  • 应用治理层权限策略
  • 识别调用者角色（公众 / 监管）
  • 遵循最小公开原则
  • 按角色过滤返回数据字段
SCENARIOS

基于调用者角色的差异化数据返回

场景 A 调用者 = 公众 (Public)

最小公开视图

  • ✅ 运行识别标识 (DET)
  • ✅ 证书信息
  • ✅ 哈希指纹
  • ✅ 有效期
  • ✅ 最小基础状态
  • ❌ 个人数据
  • ❌ 精确控制站位置

场景 B 调用者 = 监管/应急 (Regulatory)

受控扩展视图

  • ✅ 全部公众可见字段
  • ✅ 实时精确位置
  • ✅ 所有人身份证号
  • ✅ 授权依据
  • ✅ 状态信息
  • ✅ 辅助证据
  • ⚠ 需授权查询凭证
